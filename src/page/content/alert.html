Alert

사용자에게 중요한 정보나 상태를 전달하는 Alert 컴포넌트입니다. 다양한 상황에 맞는 시각적 피드백을 제공합니다.

Options

Accordion 클래스 생성자에 전달할 수 있는 옵션에 대한 자세한 설명입니다.

옵션명 타입 기본값 설명
id String 필수 아코디언 인스턴스의 고유 식별자입니다. 이는 HTML의 주 아코디언 컨테이너 요소에 있는 data-accordion 속성과 일치해야 합니다.
expanded string 또는 null null 컴포넌트 로드 시 어떤 아코디언 항목이 초기에 확장되어야 하는지 지정합니다. 확장하려는 아코디언 버튼의 ID를 제공해야 합니다. null로 설정하면 기본적으로 어떤 항목도 확장되지 않습니다.
singleOpen boolean true 한 번에 하나의 아코디언 항목만 열릴 수 있는지(true) 또는 여러 항목이 동시에 열릴 수 있는지(false)를 결정합니다. true인 경우, 한 항목을 열면 현재 열려 있는 다른 항목이 자동으로 닫힙니다.
scrollIntoView boolean false 스크롤이벤트
scrollOptions object false { behavior: 'smooth', block: 'start' }

Accessibility

아코디언에 접근 가능한 제목을 제공한다.

아코디언 헤더 내부의 제목은 페이지의 다른 콘텐츠의 구조에 맞추어 <h1>~<h6> 태그로 마크업한다. 스크린 리더 사용자가 아코디언 목록에서 어떤 정보를 확인할 수 있는지 개요를 파악하고 필요한 정보를 빠르게 탐색하는 데 도움이 된다.

  • KWCAG 2.2 제목 제공
  • WCAG 2.1 Info and Relationships (A)

아코디언의 활성화 상태 정보를 스크린 리더에서 확인할 수 있도록 한다.

아코디언의 확장 상태에 대한 정보를 텍스트 정보로 제공(펼치기/접기, 축소됨/확장됨)하여 스크린 리더 사용자가 컨트롤의 상태를 인지할 수 있도록 해야 한다.

  • WCAG 2.1 Name, Role, Value (A)

헤더를 버튼 역할로 제공한다.

헤더를 실행하였을 때 특정 섹션이나 화면으로 이동하는 것이 아니므로 <button>을 사용하거나 role="button"을 사용하여 스크린 리더에서 버튼 역할임을 인지할 수 있도록 한다.

  • WCAG 2.1 Name, Role, Value (A)
