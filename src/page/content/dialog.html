Dialog

Dialog 컴포넌트는 웹 애플리케이션에서 사용자에게 정보를 제공하거나, 입력을 받거나, 특정 작업을 확인하는 등 다양한 상호작용을 위한 팝업 UI를 쉽게 구현할 수 있도록 설계되었습니다. 이 컴포넌트는 모달(Modal), 시스템(System), **토스트(Toast)**와 같은 여러 유형의 다이얼로그를 지원하며, 접근성(ARIA)과 사용자 경험(UX)을 고려하여 개발되었습니다.

Code

HTML


//dropdown button
<div data-dropdown="dropdown1">
	<button type="button" class="btn-base" data-dropdown-button="dropdown1">
		<span data-dropdown-text="dropdown1">주요기능</span>
	</button>
</div>

//dropdown content
<div data-dropdown-panel="dropdown1" aria-hidden="true">
  <div data-dropdown-section="dropdown1">
		...
  </div>
</div>

javascript


<script type="module">
	import Dropdown from './component/dropdown.js';

	const drop1 = new Dropdown({
		id: 'userMenu',
		//헤더 영역에 드롭다운 패널 삽입
		area: document.querySelector('area-dropdown[data-area="body"]'), 
		//외부 파일에서 메뉴 콘텐츠 로드
		src: './html/user-menu.html', 
		//버튼의 오른쪽 아래에 패널 위치
		ps: 'br', 
		srcCallback: () => {
			console.log('User menu content loaded!');
			//로드된 콘텐츠 내의 특정 버튼에 이벤트 리스너 추가 등
		},
		callback: () => {
			console.log('Dropdown initialized!');
		}
	});

	//실행
	drop1.init();
	//선택 열림
	drop1.show();
	//선택 닫힘
	drop1.hide();
	//드랍다운 구성 삭제
	drop1.destroy();
</script>

Options

아래는 사용 가능한 옵션 목록입니다.

modal

옵션명 타입 기본값 설명
id String (필수) 고유한 식별자이므로 다이얼로그를 생성할 때 반드시 지정해야 합니다.
type String 'modal' 다이얼로그의 유형을 지정합니다. 'modal', 'system', 'toast'를 사용할 수 있습니다.
ps String 'center' 다이얼로그가 나타나는 위치를 설정합니다. 'center', 'top', 'bottom', 'left', 'right', 'full'이 있습니다.
move Boolean true true일 경우, 다이얼로그를 드래그하여 이동시킬 수 있습니다. ps가 'center'일 때만 유효합니다.
className String '' 다이얼로그 최상위 요소에 추가할 CSS 클래스입니다.
area DOM Element .area-dialog 다이얼로그가 추가될 부모 요소를 지정합니다.
focus_back DOM Element null 다이얼로그가 닫힌 후 포커스를 되돌릴 요소를 지정합니다.
dim Boolean true true일 경우, 다이얼로그 뒤에 어두운 배경(dimmed layer)이 생성됩니다.
dimClick Boolean true true일 경우, 배경을 클릭하면 다이얼로그가 닫힙니다.
full Boolean false true일 경우, 다이얼로그가 전체 화면으로 표시됩니다.
src String null 다이얼로그 내부에 표시할 외부 HTML 파일의 경로입니다. modal 타입에서 사용됩니다.
load String 'pre' 다이얼로그를 미리 불러올지 설정합니다. 'pre', 'lazy'
loadCallback Function null 다이얼로그 콘텐츠 로딩이 완료된 후 실행되는 콜백 함수입니다.
extend Boolean true true일 경우, 다이얼로그를 드래그하여 확장/축소할 수 있는 기능을 활성화합니다. 모바일 환경에서 유용합니다.

system

옵션명 타입 기본값 설명
title String null 다이얼로그의 제목을 설정합니다. system 타입에서 사용됩니다.
confirmText String '' 확인 버튼의 텍스트를 지정합니다. system 타입에서 사용됩니다.
cancelText String '' 취소 버튼의 텍스트를 지정합니다. system 타입에서 사용됩니다.
confirmCallback Function null 확인 버튼 클릭 시 실행되는 콜백 함수입니다.
cancelCallback Function null 취소 버튼 클릭 시 실행되는 콜백 함수입니다.
message String '' toast나 system 타입에서 표시할 메시지 내용을 설정합니다.

toast

옵션명 타입 기본값 설명
toastRole String 'status' toast 메시지의 중요도에 따라 role="status"는 "상태 정보"라는 명확한 의미를 전달, role="alert"는 "긴급 알림"이라는 의미를 전달합니다.
delay String 'short' toast 타입에서 메시지가 표시되는 시간을 설정합니다. 'short' (2초), 'long' (3.5초) 중 선택할 수 있습니다.
toastMessage String '' toast나 system 타입에서 표시할 메시지 내용을 설정합니다.

Accessibility

드랍다운을 버튼 역할로 제공한다.

드랍다운을 실행하였을 때 특정 섹션이나 화면으로 이동하는 것이 아니므로 <button>을 사용하거나 role="button"을 사용하여 스크린 리더에서 버튼 역할임을 인지할 수 있도록 한다.

  • WCAG 2.1 Name, Role, Value (A)

스크린 리더에서 확장/축소 상태 정보를 확인할 수 있도록 한다.

aria-expanded 속성이나 title 속성을 활용하여 드랍다운 섹션의 활성화 상태 정보가 스크린 리더에 탐지될 수 있도록 해야 한다.

  • WCAG 2.1 Name, Role, Value (A)
