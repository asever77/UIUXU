Tab

Tab 컴포넌트는 콘텐츠를 탭 형태로 구성하고 표시하는 유연하고 접근성 높은 방법을 제공합니다. 이 컴포넌트는 정적 및 동적 콘텐츠 로딩을 모두 지원하여 다양한 사용 사례에 적합합니다. 이 가이드를 통해 Tab 컴포넌트의 기능, 설정 방법, 그리고 사용 가능한 옵션에 대해 자세히 알아보세요.

Code

HTML


//동적 탭
<div data-tab="tab1"></div>
<div data-tab-wrap="tab1"></div>

//정적 탭
<div data-tab="tab2">
	<div role="tablist">
		<button type="button" role="tab">
			<span>탭1</span>
		</button>
	</div>
</div>
<div data-tab-wrap="tab2">
	<div role="tabpanel">
		<section>
		...
		</section>
	</div>
</div>

javascript


<script type="module">
	import Tab from './component/tab.js';
	
	const myTab = new Tab({
		//공통
		id: 'tab1',
		selected: 1, //1부터 시작

		//동적: 개별
		data: [
			{ 
				src: './tab/tab1.html', 
				tab: '<span>tab1</span>', 
				callback: () => {...},
			},
		],

		//동적: 전체
		loadAll: {
			src: './tab/tab_group.html',
		},
		data: [
			{ 
				tab: '<span>tab2-1</span>', 
				callback: () => {...},
			},
		],
		
		//정적
		renderMode: 'static',
		data: [
			{ 
				callback: () => {...},
			},
		],
	});

	//실행
	myTab.init();
	//탭 열기
	myTab.expanded('tab1-id-1');
</script>

Options

Tab 클래스 생성자에 전달할 수 있는 옵션에 대한 자세한 설명입니다.

옵션명 타입 기본값 설명
id String 필수 탭 컴포넌트의 필수 고유 식별자입니다. 이 id는 DOM에서 메인 탭 컨테이너 (data-tab="{id}")와 탭 패널 래퍼 (data-tab-wrap="{id}")를 선택하는 데 사용됩니다. 또한 접근성을 위한 ARIA 속성 (id, aria-controls, aria-labelledby)을 생성하는 데도 사용됩니다.
renderMode string 'dynamic' 탭 패널이 렌더링되고 로드되는 방식을 결정합니다.
- 'dynamic': (기본값) 각 탭에 해당하는 콘텐츠가 탭 클릭 시 개별적으로 로드됩니다 (단, loadAll이 true인 경우는 제외).
- 'static': 탭 패널의 콘텐츠가 초기 HTML에 이미 존재한다고 가정합니다. 동적으로 콘텐츠를 로드하지 않습니다.
loadAll boolean 또는 object false renderMode가 'dynamic'일 때 탭 패널 콘텐츠의 로드 방식을 제어합니다.
- false: 초기 선택된 탭의 콘텐츠만 로드됩니다 (또는 개별 탭 클릭 시 로드).
- { src: 'path/to/content.html' }: 초기화 시 지정된 src 파일에서 모든 탭 패널 콘텐츠가 한 번에 로드됩니다.
selected number 1 Tab 컴포넌트 초기화 시 어떤 탭이 활성화되어 표시될지 지정합니다. 이 값은 탭의 1부터 시작하는 인덱스에 해당합니다.
scroll boolean false true 이면 탭 패널을 전체 나열합니다. 클릭하면 해당 패널로 스크롤 이동합니다.
scrollOffsetTop Number 0 상단에서 기입한 숫자의 위치만큼 떨어진 곳에서부터 콜백을 보냅니다.
data Array<object> [] 각 탭과 그에 연결된 콘텐츠/콜백을 나타내는 객체의 배열입니다. 배열 내 각 객체는 다음을 포함할 수 있습니다:
- tab: (string) 탭 버튼에 표시될 텍스트 콘텐츠입니다.
- src: (string, renderMode가 'dynamic'이고 loadAll이 false일 경우 필수) 탭 패널에 로드될 콘텐츠의 URL입니다.
- callback: (function) 이 특정 탭이 활성화될 때 실행될 함수입니다.

Accessibility

탭의 선택 상태를 색상으로만 구분하지 않는다.

탭의 선택 상태를 색상 이외의 수단으로 구분할 수 있는 시각적 단서를 제공해야 한다. 밑줄 제공, 1px 이상의 테두리 차이, 식별자 제공 등의 방법으로 크기나 형태 차원에서 정보를 구분하는 방법을 사용할 수 있다.

  • KWCAG 2.2 색에 무관한 콘텐츠 인식
  • WCAG 2.1 Use of Color (A)

탭을 키보드로 탐색하고 실행할 수 있도록 한다.

탭은 키보드로 포커스를 이동할 수 있어야 하며, Enter 또는 Space 키를 사용하여 탭을 활성화할 수 있어야 한다. 이를 통해 키보드 사용자도 탭 인터페이스를 완전히 활용할 수 있다.

  • KWCAG 2.2 키보드 사용 보장
  • WCAG 2.1 Keyboard (A)
  • WCAG 2.1 No Keyboard Trap (A)

탭의 키보드 초점이 명확하게 표시되도록 한다.

탭이 키보드 포커스를 받을 때 시각적으로 명확하게 표시되어야 한다. 기본 브라우저 포커스 스타일을 사용하거나, 사용자 정의 스타일을 적용하여 포커스가 어디에 있는지 쉽게 알 수 있도록 해야 한다.

  • KWCAG 2.2 초점 이동
  • WCAG 2.1 Focus Visible (AA)
  • WCAG 2.1 Non-text Contrast (AA)

탭의 선택 상태를 스크린 리더로 확인할 수 있도록 한다.

탭 레이블과 탭의 선택 상태를 텍스트로 제공하여 스크린 리더에서 정보를 인지할 수 있도록 해야 한다. 탭 버튼에 role="tab"로 역할을 부여하고 선택된 탭에 aria-selected="true"로 상태 정보를 전달하거나 title="현재 탭", title="선택됨"으로 선택 상태에 대한 정보를 제공할 수 있다.

  • KWCAG 2.2 적절한 대체 텍스트 제공
  • WCAG 2.1 Name, Role, Value (A)

탭과 패널의 역할 및 관계를 스크린 리더로 확인할 수 있도록 한다.

탭 버튼 목록 영역은 role="tablist", 탭 버튼은 role="tab", 탭 패널은 role="tabpanel"로 역할을 부여한다. 그런 다음 탭 버튼과 탭 패널에 각각 id 속성과 속성값을 부여한 다음, 탭 버튼에는 aria-controls="[상응하는 탭 패널 id 속성값]", aria-labelledby="[상응하는 탭 버튼 id 속성값]"을 설정하여 탭과 패널 간 연결 관계를 프로그램적으로 전달할 수 있다.

  • KWCAG 2.2 콘텐츠의 선형화
  • WCAG 2.1 Info and Relationships (A)
  • WCAG 2.1 Name, Role, Value (A)
